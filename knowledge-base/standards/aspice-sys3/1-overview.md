# ASPICE SYS.3 — Overview

## What is SYS.3?

ASPICE SYS.3 (System Architectural Design) defines how system requirements are allocated to architectural elements and how the system structure is documented.

## Key Outputs

- System architectural design document
- Allocated requirements (bidirectional traceability to SYS.1 and SWE.1)
- Interface specifications

## Position in ASPICE Flow

```
SYS.1 (Requirements Elicitation)
    ↓
SYS.2 (System Requirements Analysis)
    ↓
SYS.3 (System Architectural Design)  ← THIS REPO
    ↓
SWE.1 (Software Requirements Analysis)
```
